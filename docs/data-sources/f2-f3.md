# Data Source Research — Formula 2 & Formula 3

**Series IDs:** `f2`, `f3`
**Status:** `integrated`
**Integration path:** `seed`
**Last reviewed:** 2026-03-22

---

## Series Overview

FIA Formula 2 and Formula 3 are the two main feeder series to Formula 1. Both support series race at the same venues as F1 during F1 race weekends (with a few exceptions). ~14 rounds per season for F2, ~10 for F3. F2 weekend: Practice, Qualifying, Sprint Race (Saturday), Feature Race (Sunday). F3 weekend: Practice, Qualifying, Race 1 (Saturday), Race 2 (Sunday).

---

## APIs & Sources Researched

No dedicated public APIs were found for either series. Both are hosted on `fiaformula2.com` / `fiaformula3.com` but no JSON endpoints have been identified.

### Observation: F1 calendar as proxy

Since F2 and F3 race at F1 venues, the F1 calendar (Jolpica API) gives the venue and weekend dates. Session times differ from F1 and would need to be added manually, but circuits and dates can be derived.

**Status:** Not currently implemented — seed files used instead as it's simpler.

---

## Decision

**Chosen source:** Manual seed files
**Integration path:** Seed (`data/seed/f2.json`, `data/seed/f3.json`)

**Rationale:**
No free public APIs found. Both series co-locate with F1, so schedules are predictable and announced alongside the F1 calendar. ~14 rounds per season — manual curation is low cost. F1 calendar could theoretically be used as a base for circuit/date data, but the complexity of syncing isn't worth it given the small number of rounds.

**Trade-offs accepted:**
- Manual update required each season
- Session times are approximate (based on standard F2/F3 slot patterns around F1 sessions)
- A few non-F1-weekend rounds (F3 has some standalone rounds in some seasons) must be manually tracked

---

## Data Mapping

Seed files are written directly in silver schema. Key notes:
- `seriesId`: `"f2"` or `"f3"` (not `"formula2"` etc.)
- Session types for F2: `"Practice"`, `"Qualifying"`, `"Sprint Race"`, `"Feature Race"`
- Session types for F3: `"Practice"`, `"Qualifying"`, `"Race 1"`, `"Race 2"`
- Circuits are identical to F1 — reuse the same `circuit` objects where possible
- `countryCode` must match F1 for the same venue (both must use alpha-2)

---

## Future Option: Derive from F1 calendar

If this becomes hard to maintain, an alternative approach:
1. After running the F1 fetcher, generate F2/F3 seed events from the F1 silver data
2. Strip F1-specific sessions, add standard F2/F3 session slots at known offsets
3. This would make F2/F3 automatically follow F1 calendar changes

Not implemented yet — seed files are simpler for now.

---

## Maintenance

- **Update frequency:** Once per season (announced alongside F1 calendar)
- **Season rollover:** Manually update `data/seed/f2.json` and `data/seed/f3.json`
- **Owner:** Manual — check `fiaformula2.com` and `fiaformula3.com` for official calendars
